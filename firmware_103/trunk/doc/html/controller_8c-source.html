00030  \mainpage MAKE Controller Kit - C API Reference
00031  
00032  \section overview Overview
00033   
00034   The MAKE Controller Board is a powerful, flexible, and easy to use microcontroller, based on the Atmel ARM7 SAM7X256.
00035   The range of applications it can perform  by itself is rather limited, however.  It is designed to interface easily to 
00036   additional circuitry in order to actually connect with the real world.
00037   
00038   The MAKE Controller Board can be plugged into into the MAKE Application Board which makes use of nearly all the 
00039   devices on the Controller or it can, alternatively, be plugged into a piece of hardware of your creation.
00040     
00041   To ensure that the software environment for the Controller is consistent, regardless of the interface hardware,
00042   the code for the Controller Board has been kept separate from the code for the Application Board to make this as easy
00043   as possible:
00044   - The <b>ControllerBoard</b> module provides the core routines required for the MAKE Controller Board.
00045   - The <b>AppBoard</b> module provides routines that work with the MAKE Application Board, and are often based on or use
00046   code from the ControllerBoard module.
00047   
00048   \subsection soft_env Software Environment 
00049   The Make Controller Kit software environment is a fusion of four main bodies of code:
00050   - User code - the applications that you, the user, create.
00051   - MakingThings library - code that MakingThings has contributed to make it easier to work with the MAKE Controller Kit.
00052   - FreeRTOS - an open source Real Time Operating System for the SAM7X.  See http://www.freetros.org
00053   - LwIP - a complete and open source TCIP/IP code library.  See http://www.sics.se/~adam/lwip/
00054 
00055   These bodies of code represent tens of 1000's of hours of work by dedicated people.  They don't always look alike.
00056 
00057  \section philosophy Philosophy
00058 
00059  The MAKE Controller Kit is a community project, designed to be rewarding to different groups:
00060   - Fabricators
00061   - Coders
00062   - Electrical Engineers
00063   - Students
00064   
00065   and everybody in between.  The idea is to get beginners to start off as simply as possible with the real thing,
00066   so they develop real skills, and provide a full-featured system & environment for more experienced developers.
00067  
00068 */
