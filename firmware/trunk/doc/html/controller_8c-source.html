00030 /**
00031  \mainpage Make Controller Kit - Firmware API
00032  
00033  \section overview Overview
00034   
00035   The Make Controller Board is a powerful, flexible, and easy to use microcontroller, based on the Atmel ARM7 SAM7X256.
00036   It is designed to interface easily to additional circuitry in order to connect with the real world.
00037   
00038   The Make Controller Board can be plugged into into the Make Application Board which makes use of nearly all the 
00039   devices on the Controller. Alternatively, it can be plugged into a piece of hardware of your creation.
00040 
00041   To jump right into browsing the API, click on \b Modules above.
00042     
00043   \section soft_env Software Environment 
00044   The Make Controller Kit software environment is a fusion of four main bodies of code:
00045   - User code - the applications that you, the user, create.
00046   - MakingThings library - code that MakingThings has contributed to make it easier to work with the Make Controller Kit.
00047   - FreeRTOS - an open source Real Time Operating System for the SAM7X.  See http://www.freertos.org
00048   - lwIP - a complete and open source TCIP/IP code library.  See http://www.sics.se/~adam/lwip/
00049 
00050   To ensure that the software environment for the Controller is consistent, regardless of the interface hardware,
00051   the code for the Controller Board has been kept separate from the code for the Application Board to make this as easy
00052   as possible:
00053   - The \ref Controller module provides the core routines required for the Make Controller Board.
00054   - The \ref AppBoard module provides routines that work with the Make Application Board, and are often based on or use
00055   code from the ControllerBoard module.
00056   - The \ref OSC module provides a communication interface to the Make Controller Kit from other computers and devices.
00057 
00058   Software for the Make Controller Kit is open source, and thus subject to the licensing of each module of code.  Be sure to acquaint 
00059   yourself with the licenses for the main bodies of code in the Make Controller firmware codebase:
00060   - MakingThings: http://www.apache.org/licenses/LICENSE-2.0.html
00061   - FreeRTOS: http://www.freertos.org/a00114.html
00062   - lwIP: http://www.xfree86.org/3.3.6/COPYRIGHT2.html#5
00063 
00064   \section Tools
00065   The firmware for the Make Controller Kit can be built using the freely available ARM-flavored version of the GCC compiler, \b arm-elf-gcc.
00066   Many toolchains are available - OS X and Windows versions can be found on the MakingThings downloads page: http://www.makingthings.com/resources/downloads
00067   Other good options include:
00068   - GNU-ARM: http://www.gnuarm.com
00069   - WinARM (Windows-only): http://www.siwawi.arubi.uni-kl.de/avr_projects/arm_projects/#winarm
00070   - YAGARTO (Windows-only): http://www.yagarto.de
00071 
00072   To upload new firmware to your Make Controller, you have a few options:
00073   - Use \b mchelper, Make Controller Helper, an app from MakingThings: http://www.makingthings.com/resources/downloads
00074   - Use \b sam7utils, a command line app: http://oss.tekno.us/sam7utils
00075   - Use a JTAG device.
00076 
00077   Rowley also provides a very good IDE for ARM7 projects called \b CrossWorks, which supports in-circuit debugging.  There are single user
00078   licenses available.  Check http://www.rowley.co.uk/arm/index.htm
00079 
00080   \section Community
00081   As you begin to create projects with the Make Controller, please consider contributing any potentially helpful material you generate
00082   or come across.  This could be anything from source code, to schematics, to instructions on how to interface with a particular device.
00083 
00084   The best places to get in touch are:
00085   - MakingThings forums: http://www.makingthings.com/forum
00086   - MakingThings bug tracker & development wiki: http://dev.makingthings.com
00087   - MakingThings IRC channel: \b makingthings on \b irc.freenode.net
00088 
00089   We look forward to hearing about your projects!
00090 */
